0. 设计思维导论 Design Thinking

什么是设计思维？

设计思维是一种以人为本的创新方法论。它不只是设计师的专属工具，而是任何希望解决复杂问题的人都能使用的方法。

核心原则：
  • • 以人为本：从用户需求出发，而非技术或商业目标
  • • 问题导向：先找到正确的问题，再寻找解决方案
  • • 迭代思维：通过不断测试和学习来优化方案
  • • 跨学科协作：融合不同视角，产生创新想法

案例分析：Airbnb 的转型

Airbnb 如何通过设计思维从一个濒临破产的公司成长为估值千亿的独角兽？

1
共情：发现根本问题

2008年，Airbnb濒临破产。创始人发现房东照片质量差，导致用户不信任。这不是技术问题，而是体验问题。

2
定义：聚焦体验提升

问题定义：如何让租客在预订前建立对房东的信任？解决方案不是复杂的功能，而是更好的照片。

3

创始人亲自上门为房东拍照，发现专业照片让订单量提升2-3倍。规模化后推出专业摄影服务。

4
测试与迭代：持续优化

通过数据验证假设，不断改进用户体验。如今Airbnb已成为全球最大的短租平台。

1. 设计心理学基础 Design Psychology

为什么需要心理学？

设计心理学帮助我们理解用户为什么会这样思考和行动。掌握这些原理，可以让你的设计更符合人类认知规律，减少用户的认知负担。

核心观点：
"设计不是艺术，而是科学——它必须遵循人类认知的规律"

1️⃣ 格式塔心理学 Gestalt Psychology

核心原则：整体大于部分之和。人类大脑会自动将分散的元素组织成有意义的整体。

接近性原则
Proximity

距离近的元素被视为一组。UI中用间距划分内容层次。

相似性原则
Similarity

相似的元素被视为一组。用颜色、形状、大小区分类别。

闭合性原则
Closure

大脑会自动补全不完整的图形。用于引导视觉流向。

⌖
连续性原则
Continuity

眼睛会沿着平滑的路径移动。引导用户关注重点。

简单性原则
Simplicity

大脑倾向于简单、对称、有序的布局。避免混乱复杂。

⬜
主体与背景
Figure-Ground

大脑会区分主体和背景。用对比突出重点元素。

2️⃣ 菲茨定律 Fitts' Law

核心原理：目标越大、距离越近，操作越容易。

时间 = a + b × log₂(D/W + 1)
📏 D = 距离 Distance

鼠标移动到目标中心的距离

📐 W = 宽度 Width

目标的大小（按钮的可点击区域）

设计实践
  • 重要按钮要大：提交、购买等核心操作使用大按钮（48px+）
  • 靠近相关元素：复选框靠近文本标签，减少鼠标移动
  • 屏幕边缘利用：重要操作放在屏幕四角（Fitts定律特例，边缘距离无限小）
  • 避免可点击区域过小：移动端按钮最小44×44px，避免误触

3️⃣ 冯·雷斯托夫效应 Von Restorff Effect

核心原理：独特的元素更容易被记忆。

在一系列相似元素中，那个不同的元素会被记住。这个效应也被称为"隔离效应"。

● ● ● ● ● ● ● ●
独特的粉色圆点更容易被记住
设计实践
  • CTA按钮突出：用对比色让主按钮与众不同
  • 重要信息强调：用颜色、大小、位置区分重要性
  • 避免过度使用：如果所有元素都独特，就都不独特了
  • 引导视觉流向：用独特元素引导用户注意

2. 情感化设计 Emotional Design

什么是情感化设计？

情感化设计不只关注"好用"，更关注触动用户的情感。Don Norman提出情感化设计有三个层次，每个层次都影响用户的体验。

核心观点：
"人类是情感动物，设计不能只关注功能，更要触动人心"

Don Norman 情感化设计三层次

👀
本能层
Visceral

用户第一眼看到设计时的即时反应。这是感性、直觉的层次。

影响因素
  • • 外观美感
  • • 色彩搭配
  • • 质感材质
  • • 视觉冲击力
设计要点
  • 精美的视觉效果
  • 和谐的配色方案
  • 吸引人的动画
  • 高质量的产品图
行为层
Behavioral

用户使用过程中的体验。这是功能、性能、可用性的层次。

影响因素
  • • 功能完整性
  • • 操作流畅度
  • • 响应速度
  • • 易学易用
设计要点
  • 流畅的交互体验
  • 即时的反馈
  • 清晰的导航
  • 高效的任务完成
🤔
反思层
Reflective

用户使用后的回忆和评价。这是理性、反思的层次。

影响因素
  • • 品牌价值
  • • 使用记忆
  • • 自我认同
  • • 情感连接
设计要点
  • 品牌故事传递
  • 个性化体验
  • 惊喜和愉悦
  • 社交认同

情感化设计技巧

微动画 Micro-animations

通过细腻的动画传递情感，让界面更有生命力。

示例：按钮点击时的弹跳、加载时的动效、成功时的庆祝动画
💬
个性化反馈 Personalized Feedback

根据用户行为提供个性化的反馈和鼓励。

示例：Duolingo的"你真棒"、Spotify的"今日回顾"
惊喜元素 Surprise & Delight

在关键时刻给用户小惊喜，创造难忘体验。

示例：Google搜索的小彩蛋、Airbnb的"已预订"动画
🌈
情感化文案 Emotional Copy

用有温度的语言和用户沟通，建立情感连接。

示例：错误提示时用"哎呀"代替"Error"

真实案例分析

Notion 的欢迎动画

首次打开Notion时，会看到一个可爱的欢迎动画——小方块跳跃、堆叠，形成Notion的logo。这个简单的动画在"本能层"打动用户，传达了Notion的友好和创造感。

Duolingo 的错误提示

当用户答错题目时，Duolingo不会冷冰冰地说"错误"，而是用可爱的角色表情和鼓励的话语，如"没关系，下次一定能对"。这种"行为层"的反馈降低了挫败感。

Spotify 的年度回顾

每年底的Spotify Wrapped通过精美的数据可视化、个性化的音乐回忆，让用户感受到"我这一年听音乐的故事"。这是"反思层"的杰作，建立了深厚的品牌情感连接。

用户研究是系统地收集和分析用户信息，以理解用户需求、行为和痛点。它是设计思维的起点，没有用户研究，设计就成了"自嗨"。

6. 同理心地图 Empathy Mapping

什么是同理心地图？

同理心地图是从用户视角思考的框架工具。它通过四象限结构，帮助设计师深入理解用户的需求、痛点、动机和情绪。

核心价值：
超越"做什么"，深入理解"为什么"

四象限结构

💬
说什么 Says

用户口头表达的内容、直接陈述的观点和需求。

示例
  • "这个功能太难用了"
  • "我希望有个快速分享的按钮"
👆
做什么 Does

用户的实际行为、操作习惯、使用模式。

示例
  • 每天打开App 3次
  • 只在晚上使用
想什么 Thinks

用户的内心想法、假设、疑问和顾虑。

示例
  • "这个功能安全吗？"
  • "我是不是用错了？"
😊
感受如何 Feels

用户的情绪体验、情感变化、感受和态度。

8. 视觉层次：引导注意力

视觉层次决定了用户阅读信息的顺序。通过大小、颜色、对比度、位置来控制注意力流向。

理论基础：格式塔心理学

格式塔（Gestalt）是20世纪初德国心理学家提出的理论，揭示了人类视觉如何自动组织和理解图形。 这些原理是视觉设计的科学基础——理解它们，你就知道「为什么」某些设计有效。

📐 格式塔五大原理

1. 接近性 Proximity

靠近的元素被视为一组。这就是为什么信息架构强调「就近原则」

2. 相似性 Similarity

相似的元素被视为一组。这是配色和图标风格统一的理论依据

3. 连续性 Continuity

眼睛会沿着线条或曲线自然移动。用于引导视觉流

4. 闭合性 Closure

大脑会自动补全不完整的形状。极简 Logo 设计常用此技巧

5. 图底关系 Figure-Ground

人会区分前景与背景。这是玻璃态和卡片设计的原理

Disney 动画 12 原则（UI 适用精选）

迪士尼动画师在1981年总结的 12 条动画原则，至今仍是动效设计的理论基础。以下 4 条对 UI 最为实用：

缓动 Easing

动作起止要有加速/减速，避免匀速的机械感。（ease-in-out）

预备动作 Anticipation

动作前的小反向运动，增加真实感。（按钮按下时微缩）

跟随动作 Follow Through

主体停止后，附属部分继续运动。（弹性回弹效果）

次要动作 Secondary Action

主动作之外的辅助动画，增强表现力。（图标微动）

    34. 移动端 UI 设计

    Fitts' Law 费茨定律

    1954年心理学家 Paul Fitts 提出：点击目标所需时间与距离成正比，与目标大小成反比。 这解释了为什么移动端按钮要足够大、重要操作要放在容易触及的位置。

    T = a + b × log₂(1 + D/W)
    T=时间 | D=距离 | W=目标宽度
    最小点击区域 44×44px
    Apple HIG 推荐标准

    CTA 按钮比普通按钮大

    拇指热区理论

    Steven Hoober 的研究显示，75% 的用户单手持机、用拇指操作。屏幕可分为三个区域：

    难以触及
    需伸展
    舒适区

    37. AI 提示词指南

    核心原则：明确风格 + 具体描述 + 反面排除 + 系统约束

    好的提示词 = 告诉 AI 你要什么 + 告诉 AI 你不要什么 + 定义设计系统约束

    ⚠️ 警惕 "AI Slop" 美学

    "AI Slop" 指 AI 生成的泛化、无特色的设计输出。如果你不加约束地让 AI 设计前端，往往会得到：

    ✗ 紫蓝渐变 + 圆角卡片 + 随机emoji
    ✗ Tailwind 默认色 blue-500、gray-100
    ✗ 过度使用 Heroicons 装饰
    ✗ 无层次的信息架构

    🎛️ 四维控制法：约束 AI 输出

    好的 AI 前端设计需要在四个维度给出明确约束，而不是让 AI 自由发挥：

    1. 字体 Typography

    ✓ 指定具体字体: "Use Inter for body, Space Grotesk for headings"

    ✗ 让 AI 自选: 会得到 Arial、system-ui

    2. 配色 Color

    ✓ 定义调色板: "Primary #2563EB, accent #F59E0B, neutral slate"

    ✗ 不指定: 会用 blue-500、indigo-600 等默认色

    3. 动效 Motion

    ✓ 说明动效策略: "Subtle transitions, 200ms duration, ease-out"

    ✗ 不提及: 要么没动效，要么过度动效

    4. 背景/纹理 Background

    ✓ 明确背景: "Clean white, subtle grid pattern, no gradients"

    ✗ 不指定: 随机渐变或纯白无层次

    🐛 5 个常见 AI 前端错误及提示词修复

    错误 1：图标乱用/图标风格不一致

    AI 会随意添加装饰性图标，或混用不同风格（线性+填充+emoji）。

    # 修复提示词
    "Use icons from Lucide React only. Style: stroke-width 1.5, 20px size.
    Only use icons for: navigation, actions, status indicators.
    Never use decorative icons or emoji."
    错误 2：无脑 Tailwind 默认配色

    AI 默认使用 blue-500、gray-100 等，导致所有项目看起来一样。

    # 修复提示词
    "Do NOT use default Tailwind colors.
    Use custom palette: primary #1E40AF, secondary #7C3AED,
    surface #F8FAFC, text #0F172A.
    Reference: 60-30-10 color rule."
    错误 3：改了这里忘了那里（状态同步问题）

    修改按钮样式时，AI 可能只改一处，忘记更新 hover/disabled/loading 等状态。

    # 修复提示词
    "When modifying any component, always update ALL states:
    - default, hover, focus, active, disabled, loading
    - light mode AND dark mode variants
    - mobile AND desktop responsive versions
    List all affected components before making changes."
    错误 4：缺少完整的交互状态

    AI 只实现默认状态，忘记空状态、加载状态、错误状态等。

    # 修复提示词
    "Implement complete state coverage:
    - Empty state (no data, first-time user)
    - Loading state (skeleton, spinner)
    - Error state (with retry action)
    - Success state (with feedback)
    - Partial state (some items loaded)"
    错误 5：响应式设计不完整

    桌面端好看，手机端元素挤在一起或超出屏幕。

    # 修复提示词
    "Design mobile-first (参考移动优先原则):
    - Base styles for mobile (< 640px)
    - sm: breakpoint for tablet (640px+)
    - lg: breakpoint for desktop (1024px+)
    - Test by resizing to 375px width."

    • • 页面结构和布局
    • • 主要功能区域
    • • 导航和入口

    • • 元素的相对位置

    "原型是设计者和开发者、产品经理之间的通用语言。"

    — 可交互的原型比静态设计稿更有说服力

    🔗 Figma 原型功能详解

    Figma 的原型功能允许你创建可交互的设计稿，无需写代码就能演示用户体验。

    1 Smart Animate 智能动画

    自动计算元素位置、大小、透明度变化，制作平滑过渡。最适合组件变形动画。

    2 过渡效果 Transition

    Dissolve（渐变）、Push（推入）、Slide out（滑出）、Navigate to（页面跳转）

    3 触发器 Trigger

    On click（点击）、On hover（悬停）、On drag（拖拽）、After delay（延迟触发）

    4 叠加层 Overlay

    从边缘弹出面板，模拟移动端抽屉/侧边栏。支持从上/下/左/右弹出。

    常见交互模式实现

    🔗 页面跳转

    按钮点击 → 导航到目标页面 → Transition: Slide In / Push Right

    应用场景：登录后跳转首页、点击卡片进入详情页

    💬 模态弹窗

    按钮点击 → 显示弹窗层（背景变暗）→ Overlay: Open / Smart Animate（放大效果）

    应用场景：确认对话框、表单提交、图片预览

    📋 下拉菜单

    按钮点击 → 菜单从下方滑入 → Overlay: Bottom / Smart Animate（位移）

    应用场景：选择器、筛选器、操作菜单

    Tab 切换

    Tab 点击 → 内容区域变化 → Smart Animate（保留位置，内容淡入）

    应用场景：产品分类切换、日期选择、多标签页

    ⌨️ 键盘交互

    输入框回车 → 触发提交 / Esc 关闭弹窗 / Tab 切换焦点

    应用场景：表单提交、对话框关闭、焦点管理

    组件状态机

    在 Figma 中用 Variants 创建状态切换，如按钮的 default → hover → active。这是创建可交互原型的核心。

    Default → Hover → Active → Disabled

    每个状态对应一个 Variant，用连线定义转换逻辑（如 On hover → Hover state）

    ⏱️ 动效参数设置

    好的动效需要合适的参数。参考 动效设计章节的 Disney 动画原则。

    动画时长 Duration

    200-300ms（快速交互）、500-800ms（页面过渡）、1s+（强调效果）

    缓动函数 Easing

    Ease-out（退出）、Ease-in-out（通用）、Bounce（弹性回弹）

    黄金法则：动效时长 × 2 = 人类感知阈值。超过 200ms 的延迟才会被注意到。

    48. Landing Page 设计

    "Landing Page 唯一的目标就是转化。"

    — 用户应该在一个页面内完成你期望的行动

    Landing Page 的核心目标

    Landing Page（落地页）不是企业官网，而是一个专注于单一目标（注册、购买、下载）的营销页面。

    转化率

    让访客完成目标行为（注册/购买）

    快速决策

    减少分心，帮助用户快速做决定

    单一焦点

    页面只有一个主要 CTA（行动号召）

    Landing Page 标准结构

    一个高转化率的 Landing Page 通常包含以下7个关键部分。

    1
    Hero Section（首屏）

    大标题 + 副标题 + 主 CTA + 产品截图/视频。这是最重要的部分，3秒内抓住用户注意力。

    2
    Features（特性展示）

    3-6个核心卖点，图文结合。每个特性配一个简短描述，突出"对我有什么好处"而非功能本身。

    3
    Social Proof（社会证明）

    用户评价、Logo 墙（知名客户）、使用数据（"10,000+ 用户"）。证明产品值得信任。

    4
    Pain Point-Solution（痛点-解决方案）

    先描述用户遇到的痛苦（"每次报销要填5张表"），再展示你的方案如何轻松解决。

    5
    Pricing（价格方案）

    3个价格档位（基础版/专业版/企业版），中间的专业版用高亮突出显示（推荐）。

    6
    FAQ（常见问题）

    5-10个最常被问到的问题，用手风琴折叠显示。解答价格、退款、隐私等疑虑。

    7
    Final CTA（最终行动号召）

    页面底部再次显示主 CTA，配合紧迫感文案（"限时 7 天免费试用"）。

    Landing Page 设计技巧

    1. 清晰的视觉层级

    CTA 按钮最突出，用大字号、鲜艳颜色、留白包围。标题 > 副标题 > 正文。

    2. 单一 CTA 原则

    页面只有一个主要行动按钮。如果需要辅助链接，放在次要位置（如 FAQ）。

    3. 视觉节奏

    大标题 + 小文字 + 图片交替出现，避免单调。每个区块用不同背景色区分。

    4. 移动优先

    首屏内容在 375px 手机屏上完整显示。按钮至少 44×44px，方便点击。

    5. A/B 测试

    测试不同标题、CTA 文案、颜色。数据驱动优化，不要凭直觉。

    6. 加载速度

    首屏 < 2s 加载。压缩图片、懒加载、使用 CDN。

    优秀案例：Linear Landing Page

    Linear 的 Landing Page 被认为是 SaaS 产品的设计标杆，值得学习。

    ✦
    极简美学：黑色背景 + 白色文字 + 蓝色渐变按钮，符合品牌调性。
    ✦
    动态演示：Hero Section 使用视频展示产品界面，直观传达功能。
    ✦
    微交互：按钮悬停时渐变流动，增强高级感。
    ✦
    客户 Logo 墙：知名公司 Logo 集中展示，建立信任。
    ✦
    单一 CTA：所有引导都指向"Get Started"，避免分心。

    49. Dashboard 设计

    "Dashboard 应该让用户 10 秒内找到目标信息。"

    — 好的 Dashboard 是高效的，不是炫技的

    什么是 Dashboard？

    Dashboard（仪表盘）是数据可视化界面，用于快速浏览关键指标（KPI）、监控业务状态、做出决策。

    适用场景
    • • 电商后台（销售数据、库存）
    • • SaaS 系统（用户增长、活跃度）
    • • 分析平台（网站流量、转化率）
    • • 项目管理（进度、任务状态）
    用户需求
    • • 快速了解现状
    • • 发现异常和趋势
    • • 导向详情页面
    • • 支持数据对比

    ⚙️ Dashboard 设计原则

    1
    一屏原则

    核心数据必须在首屏可见。用户不应滚动才能看到最重要的 KPI。

    2
    层次清晰

    KPI 在最显眼位置（顶部居中或左上角），用大字号、鲜艳颜色、留白突出。

    3
    可扫描

    用户应该 10 秒内理解页面全貌。使用视觉层次、分组、留白引导视线。

    4
    可操作

    数据背后应该有行动按钮。例如，看到低转化率可以点击查看详情或导出报告。

    Dashboard 标准布局

    顶部导航
    页面切换 | 用户菜单
    侧边栏
    KPI 卡片（3-6个）
    主要图表（折线/柱状）
    详情列表 / 最近活动
    顶部导航
    • • 页面切换（Dashboard/分析/设置）
    • • 全局搜索
    • • 用户菜单（头像/通知/登出）
    侧边栏
    • • 二级导航（产品分类/时间段）
    • • 筛选器（日期/地区/渠道）
    • • 快速操作（导出/分享）
    KPI 卡片
    • • 数值 + 变化趋势（↑12%）
    • • 用颜色区分（绿=上升/红=下降）
    • • 悬停显示详情
    图表
    • • 折线图：趋势变化（时间序列）
    • • 柱状图：数据对比（分类对比）
    • • 饼图：占比（≤4 个分类）

    56. 无障碍设计基础 Accessibility

    "无障碍设计不是可选项，而是必选项。"

    — 全球 15% 人口有某种形式的残障

    ♿ 什么是无障碍设计？

    为所有用户（包括残障用户）提供可访问的设计。这不是"额外功能"，而是产品的基础要求。

    残障类型
    • • 视觉障碍：色盲、低视力、失明
    • • 听觉障碍：听力损失、失聪
    • • 运动障碍：手部不便、肌无力
    • • 认知障碍：ADHD、自闭症、阅读障碍
    影响范围
    • • 全球 15% 人口（12 亿人）
    • • 65 岁以上人口 50% 有残障
    • • 法律要求（美国 ADA、欧洲 EAA）
    • • 提升 SEO（搜索引擎更易理解）

    WCAG 2.2 标准

    Web Content Accessibility Guidelines（网页内容无障碍指南）是国际标准，包含 4 个核心原则。

    👀 可感知 Perceivable

    用户必须能感知信息和界面元素。如：图片有 alt 文本、视频有字幕。

    ⌨️ 可操作 Operable

    用户必须能用键盘完成所有操作。如：Tab 导航、Enter 激活。

    可理解 Understandable

    信息和操作必须清晰。如：表单有标签、错误提示明确。

    🛡️ 健壮 Robust

    内容必须能被辅助技术理解。如：语义化 HTML、ARIA 属性。

    无障碍设计的价值

    扩大用户群

    让 15% 的残障用户也能使用你的产品。这是巨大的市场机会。

    提升 SEO

    搜索引擎能更好地理解语义化 HTML、图片 alt 文本，提升排名。

    符合法律要求

    美国 ADA 1990、欧洲 EAA 2025、加拿大 AODA 等法律强制要求无障碍。

    社会责任

    为残障用户创造平等的机会，是企业社会责任的一部分。

    57. 屏幕阅读器支持 Screen Reader Support

    "屏幕阅读器用户依赖语义化 HTML 和 ARIA 属性。"

    — 正确的标签比精美的视觉更重要

    🎧 什么是屏幕阅读器？

    屏幕阅读器是将网页内容转换为语音的软件，供视障用户使用。

    常见屏幕阅读器
    • • NVDA（Windows，免费）
    • • JAWS（Windows，付费）
    • • VoiceOver（macOS/iOS，内置）
    • • TalkBack（Android，内置）
    用户操作方式
    • • 键盘快捷键导航
    • • 触摸手势（移动端）
    • • 语音命令

    🏷️ ARIA 属性 Accessible Rich Internet Applications

    ARIA 是一组 HTML 属性，帮助辅助技术理解复杂的交互组件。

    role：定义元素角色
    <div role="button">点击我</div>
    <nav role="navigation">...</nav>
    <div role="alert">错误提示</div>
    aria-label：为无标签元素提供描述
    <button aria-label="关闭">×</button>
    aria-describedby：关联元素与说明
    <input aria-describedby="password-help">
    <div id="password-help">至少 8 位</div>
    aria-live：声明动态内容区域
    <div aria-live="polite">动态消息</div>

    语义化 HTML

    使用正确的 HTML 标签，而不是 div + class。屏幕阅读器依赖语义化标签导航。

    ❌ 错误做法
    <div class="header">...</div>
    <div class="nav">...</div>
    <div class="main">...</div>
    正确做法
    <header>...</header>
    <nav>...</nav>
    <main>...</main>

    58. 键盘导航设计 Keyboard Navigation

    "所有交互都必须可以通过键盘完成。"

    — 这是无障碍的最低要求

    ⌨️ 为什么需要键盘导航？

    运动障碍用户

    手部不便、肌无力等用户无法使用鼠标，必须依赖键盘。

    高级用户

    键盘操作更快、更高效，很多专业用户习惯用键盘。

    🎹 键盘操作模式

    Tab / Shift+Tab

    移动焦点。Tab 向前，Shift+Tab 向后。

    Enter / Space

    激活按钮、链接、复选框。

    方向键

    在列表、菜单中移动焦点。

    Escape

    关闭模态框、取消操作。

    键盘导航设计要求

    焦点可见性 Focus Visible

    清晰显示当前焦点元素。使用 CSS :focus-visible 伪类。

    :focus-visible {
    outline: 2px solid blue;
    outline-offset: 2px;
    }
    焦点顺序 Logical Order

    符合视觉顺序。从左到右、从上到下。

    焦点陷阱 Focus Trap

    模态框打开时，焦点应限制在模态框内，无法移到背景。

    Skip Links

    提供"跳过导航"链接，让用户直接跳到主内容。

    59. 色彩对比度进阶 Color Contrast

    "对比度不只是为了好看，更是为了可读性。"

    — WCAG AA 级是最低要求

    🔢 对比度计算公式

    对比度 = (L1 + 0.05) / (L2 + 0.05)，其中 L 是相对亮度。

    // 相对亮度计算
    L = 0.2126R + 0.7152G + 0.0722B
    // 对比度计算
    ratio = (L1 + 0.05) / (L2 + 0.05)

    WCAG 2.1 标准

    AA 级（最低要求）
    • • 正常文本：4.5:1
    • • 大文本（18pt+）：3:1
    AAA 级（推荐）
    • • 正常文本：7:1
    • • 大文本：4.5:1

    什么是微交互？

    1. 触发 Trigger

    2. 规则 Rules

    系统如何响应。

    3. 反馈 Feedback

    交互的循环流程。

    ✓ 动画 + 文字提示。

    63. 高级动效技巧 Advanced Motion

    "高级动效不只是好看，更是功能性的。"

    — 每个动画都应该有明确的用途

    🎢 复杂缓动函数

    不要只用 ease-in-out，尝试自定义贝塞尔曲线创造更有趣的效果。

    自定义贝塞尔
    cubic-bezier(0.68, -0.55, 0.27, 1.55)

    弹性效果，超出边界后反弹。

    Elastic

    像弹簧一样的弹性效果。适合弹窗、按钮。

    Bounce

    多次反弹后停止。适合强调元素。

    序列动画

    元素依次出现，营造节奏感。避免所有元素同时出现。

    Staggered Animation

    元素依次出现，每个元素延迟一定时间。如：列表项依次滑入。

    延迟设计

    营造节奏感。第一个元素 0s，第二个 100ms，第三个 200ms。

    ⚙️ 物理模拟

    使用物理公式创建更自然的动画。

    弹簧 Spring

    弹簧系统，有弹性。适合抽屉、弹窗。

    阻尼 Damping

    逐渐减速，平滑停止。适合滚动、滑块。

    重力 Gravity

    模拟重力，自然下落。适合卡片掉落。

    视差滚动 Parallax

    多层背景不同速度滚动，创造深度感。注意性能，不要过度使用。

    多层背景

    背景层、中景层、前景层。不同滚动速度。

    深度感

    前景移动快，背景移动慢，产生 3D 效果。

    64. 音频交互设计 Audio Interaction

    "音频交互应该增强体验，而不是干扰。"

    — 必须提供静音选项

    音频交互类型

    音效反馈

    点击、成功、错误、通知。简短（≤200ms），不刺耳。

    语音交互

    语音搜索、语音命令。提供清晰反馈。

    音乐背景

    营造氛围。谨慎使用，必须可关闭。

    音效设计原则

    简短

    ≤200ms。过长会干扰用户。

    舒适

    不要刺耳。避免高频、尖锐的声音。

    可关闭

    用户可选择静音。不要自动播放。

    一致

    相同操作相同音效。建立用户预期。

    里程碑奖励。如"连续打卡 7 天"。

    66. 国际化设计基础 Internationalization

    阿拉伯文、希伯来文、波斯文等语言从右向左阅读。

    direction: rtl;
    镜像元素

    左右箭头、图标、布局需要镜像。

    🌍 文化适配

    白色（西方婚礼 vs 东方葬礼）。注意文化差异。

    67. 设计与性能 Performance-Aware Design

    "设计决策直接影响性能。"

    — 设计师也需要考虑加载速度

    核心性能指标

    FCP 首屏加载

    首次内容绘制 ≤1.8s。用户看到第一个元素。

    LCP 最大内容绘制

    最大元素绘制 ≤2.5s。主要内容可见。

    INP 交互延迟

    首次输入延迟 ≤100ms。响应快速。

    CLS 布局偏移

    页面稳定 ≤0.1。避免元素跳动。

    设计决策对性能的影响

    1. 图片
    • • 格式：WebP > AVIF > JPEG/PNG
    • • 分辨率：2x 屏幕用 2x 图
    • • 懒加载：视口外图片延迟加载
    • • 响应式：srcset 属性
    2. 字体
    • • 格式：WOFF2 > WOFF > TTF
    • • 子集化：只包含使用字符（减少 50%+）
    • • 预加载：关键字体 <link rel="preload">
    • • 字体显示：font-display: swap
    3. 动画
    • • 优先 CSS 动画（硬件加速）
    • • 避免 Layout Thrashing（频繁读写布局）
    • • 限制动画数量 ≤2 个并发

    性能测试工具

    Lighthouse

    Chrome DevTools，综合评分。集成在浏览器中。

    WebPageTest

    多地点测试，详细分析。在线工具。

    GTmetrix

    详细分析，优化建议。在线工具。
